## 1. プロジェクト概要

Keithley 2400 / Keithley 2612B（ソースメータ）とTOPCON BM9（輝度計）を使い、太陽電池(OPV)・発光素子(LED/OLED)のIV/JV特性・輝度を計測するPyQtアプリケーションを新規開発している。既存の単発実行スクリプト(`keithley2400/`, `keithley2600/`フォルダ)をMVVMアーキテクチャのGUIアプリに再構築するプロジェクト。

- 本番環境: Python 3.9 + PyQt5(研究室PC)。開発環境: Python 3.13 + PyQt6(このマシン)。**両対応必須**。
- PEP8準拠、MVVMアーキテクチャ厳守。
- 実機がなくてもテストできるモッククラスが必須要件。

## 7. 重要な注意点・規約(繰り返しになるが重要)

- **PyQt5/6両対応**: 全GUIコードは`qtcompat.py`経由でのみQtをimportする。直接`import PyQt5`等禁止。
- **Python3.9対応**: 全モジュール先頭に`from __future__ import annotations`必須。`match`/`case`禁止。
- **PEP8準拠**、コメントは非自明な理由がある箇所にのみ短く(何をしているか自明なコードにコメント不要)。
- **MVVM**: Model層(`models/`)はQt非依存の純Python。ViewModel層はViewインスタンスを受け取りウィジェットに直接アクセスする設計(このプロジェクトの現実装パターン)。View層はロジックを持たない(ボタン等の`clicked`シグナルはViewModelが結線する)。
- **グローバルpip install禁止**。必ず`.venv313`を使う。
- **QThread.terminate()は絶対禁止**。中断は`request_stop()`による協調的フラグで行う。
- チャンネル不正指定(`smua`/`smub`以外)は実機・モック双方とも`ValueError`に統一済み(当初モックは`InstrumentError`だったが要修正・修正済み)。
- BM9の輝度値は`*100`倍して使う(既存コード`Keithley2400_JVL.py`の単位換算踏襲)。
- モードAは「2chの平均」ではなく「smuaで電圧印加、smubを0V固定の仮想接地電流計として使用し、smubの電流測定値に`-1.0`倍したものを採用」という設計(要件定義書A-4-4-1節参照、ユーザーとの合意事項)。
