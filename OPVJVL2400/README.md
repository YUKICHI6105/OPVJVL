# OPV / JVL Measurement GUI

Keithley 2400(シリアル/SCPI)を使ったOPV測定・JVL測定を、PyQt6のタブ切り替えGUIから
操作できるようにしたもの。

## 構成

```
instruments.py           Keithley2400(シリアル/SCPI)、BM9のドライバクラス
measurement_worker.py     OPV/JVL測定ループを実行するQThreadワーカー(2種)
main_gui.py                 PyQt6メインウィンドウ(エントリーポイント)、OPVTab/JVLTab
requirements.txt
```

## 実行方法

```bash
pip install -r requirements.txt
python main_gui.py
```

## 設計方針

### タブ構成
- **OPVタブ**と**JVLタブ**を切り替え可能。
- **共通なのは「Keithley2400のCOMポート」と「保存先フォルダ」の2つのみ**。
  実機のKeithleyは1台を想定しているため、ポート設定を1箇所にまとめている。
- それ以外の設定(電圧掃引条件、繰り返し回数、ディレイ時間、NPLC、
  コンプライアンス電流、電流レンジ自動設定、サンプル名)は**タブごとに完全独立**しており、
  片方のタブでの変更がもう片方に影響することはない。
- ログ表示欄も共通で1つ。

### 初期値(ベースコードの値をそのまま採用)

| 項目 | OPVタブ | JVLタブ |
|---|---|---|
| Vmin | -0.1 V | -1.0 V |
| Vmax | 1.1 V | 1.2 V |
| Vstep | 0.02 V | 0.1 V |
| 繰り返し回数(iteration) | 3 | 3 |
| ディレイ時間 | 1.0 s | 1.0 s |
| NPLC | 1.0 | 1.0 |
| コンプライアンス電流 | 0.02 A | 0.02 A |
| 電流レンジ自動 | ON | ON |
| 電流上限(早期終了しきい値) | — | 0.001 A (1 mA) |
| 輝度倍率 | — | 100倍 |

いずれの項目もGUI上で自由に変更可能(初期値はベースコードの値を反映しているだけ)。

### `integration_time`変数について

元コード(`Keithley2400_JVL.py` / `Keithley2400_OPV.py`)にあった`integration_time = 0.5`は、
実際には`configure_source_voltage()`に渡されておらず未使用でした。実質的にその役割を
担っているのは`nplc`パラメータ(`configure_source_voltage(nplc=1.0, ...)`)のため、
本GUIでは`integration_time`という変数は作らず、**NPLC入力欄に一本化**しています
(初期値1.0、GUI上で変更可能)。

### JVLの早期終了ロジック

- 電圧は`Vmin`(初期値 -1.0V)から掃引を開始する。
- **各点で電流を測定するたびに |電流| と「電流上限」を比較**し、
  上限に達した時点でその点までのデータを保持したまま掃引を打ち切る。
- Vmaxまで掃引しても上限に達しなければ、通常通りVmaxで終了する。
  (「上限到達」と「Vmax到達」のどちらか早い方で終了)
- **早期終了・ユーザーによる中断のいずれの場合も、それまでに測定済みの点は
  必ずCSVに保存される**(部分データの保存)。
- 電流上限のデフォルトは1mA(0.001A)だが、GUI上の「電流上限 |I|」欄で変更可能。

### プロット

- OPVタブ: I-V(リニアスケール、元コードと同じ)
- JVLタブ: I-V・L-Vの2枚をタブ表示、どちらも**縦軸logスケール・絶対値**
  (元コードの`ax.set_yscale("log")`、`np.abs()`表示を踏襲)。

## 動作確認について

このコード生成環境には実機のKeithley2400/BM9がないため、以下の検証を行っています。

- 全ファイルの構文チェック(`py_compile`)
- PyQt6によるGUI構築・表示(オフスクリーンモードでの起動確認)、初期値がすべて
  要件通りであることの確認
- `instruments.Keithley2400` / `instruments.BM9` をモッククラスに差し替えた上での
  ワーカーのend-to-end実行:
  - OPVワーカー: 電圧掃引が最後まで実行され、CSVが正しい列で保存されることを確認
  - JVLワーカー: ダイオード特性を模したダミー電流で、|電流|が1mAを超えた時点で
    掃引が打ち切られ、それまでの部分データがCSV保存されることを確認

実機との通信(SCPIコマンドの応答形式、BM9のタイムアウト挙動、COMポート名など)は
実際の装置構成に依存するため、お手元の環境での最終確認をお願いします。
